# 跨模块打通规格：项目文件、伦理递交、培训、财务费用

> 目标：避免“文件清单、递交包、培训记录、伦理费”各自维护一套数据，减少重复录入和版本错误。

## 核心原则

1. 项目文件库是文件名称、类型、版本、版本日期的源头。
2. 伦理递交包引用项目文件，而不是手工重复录入全部文件信息。
3. 递交包保存递交时的文件快照，避免未来项目文件更新后影响历史递交记录。
4. 培训计划引用项目文件和授权研究人员，并保存培训时的人员快照。
5. 会审/快审产生的伦理审查费应进入财务模块，不在伦理模块里另做一套费用台账。
6. Dashboard 最终要能跨模块提醒：待递交、待培训、待伦理签收、待批件、待缴费、待发票。

## 模块关系

```text
项目文件库
  ├─ 选择文件/版本 → 伦理递交包
  ├─ 触发培训要求 → 培训计划 → 授权研究人员
  │                         └─ 生成培训记录 Word → 签署/收集证据
  ↓ 会审/快审触发费用
财务管理：伦理审查费
  ↓ 状态回写
Dashboard 风险和待办提醒
```

## 项目文件管理

### 目的

集中维护项目层面文件，供伦理递交、中心文件追踪、后续 AI 检查使用。

### 建议实体：project_documents

字段建议：

- `id`
- `project_id`
- `doc_category`：文件大类，如方案、知情同意书、药品管理、中心实验室、安全性文件、其他
- `doc_name`：文件名称
- `version`：版本号
- `version_date`：版本日期
- `effective_date`：生效日期
- `received_date`：项目组发布时间/CRA 接收日期
- `requires_ethics_submission`：是否需要递交伦理
- `requires_training`：是否需要培训
- `training_scope`：培训范围
- `training_due_days`：建议多少天内完成培训
- `notes`
- `created_at`
- `updated_at`

### 使用场景

项目组发布新文件后，CRA 先录入项目文件库：

- 药品管理手册 V1.1
- 中心实验室手册 V2.0
- 方案修正案 V3.0
- 知情同意书修正版 V4.0

然后在伦理递交包中选择这些文件，而不是重新手输。

如果文件需要培训，则基于该文件创建培训计划，选择中心和授权研究人员，追踪完成情况。

## 伦理递交包与文件库打通

### 建议实体：ethics_submission_items

递交包文件清单应同时保存：

- `project_document_id`：引用项目文件
- `doc_category_snapshot`
- `doc_name_snapshot`
- `version_snapshot`
- `version_date_snapshot`
- `copies`
- `required_original`
- `notes`

### 为什么需要快照

同一个项目文件未来可能更新版本，但历史递交包必须保留当时递交的版本信息。

例如：

- 2026-07-01 递交：方案 V2.0
- 2026-08-01 项目文件库更新：方案 V3.0

历史递交包仍应显示 V2.0，不能被自动改成 V3.0。

## 项目文件库与培训管理打通

详见 `TRAINING_WORKFLOW_SPEC.md`。

当项目文件标记 `requires_training = true` 时：

- 项目详情应提示该文件需要培训。
- 可从项目文件创建培训计划。
- 培训计划选择授权研究人员作为培训对象。
- 培训记录保存人员姓名、角色、文件版本快照。
- 培训计划可生成培训记录/签到表 Word，并追踪打印、签署和证据收集。

Dashboard 后续应提醒：

- 新文件需要培训但尚未创建培训计划。
- 培训计划即将到期或已逾期。
- PI/Sub-I 等关键角色未完成培训。
- 培训证据未收集。
- 培训记录 Word 已生成但签署版未收集。

## 伦理递交包与财务打通

### 触发规则

当递交包审查方式为：

- 会审
- 快审

系统应提示是否创建伦理审查费记录。

当递交包审查方式为：

- 备案

默认不创建伦理审查费，但允许手动标记“该中心备案也收费”。

### 建议实体：finance_ethics_fees

字段建议：

- `id`
- `project_id`
- `center_id`
- `submission_package_id`
- `fee_name`：如伦理审查费、修正案审查费
- `review_method`
- `amount`
- `payment_status`：未申请、已申请、已打款、已到账、无需支付
- `invoice_status`：无需发票、待开票、已开票、已收票
- `invoice_number`
- `invoice_amount`
- `payment_date`
- `invoice_date`
- `notes`
- `created_at`
- `updated_at`

### 状态联动

伦理递交包中只显示财务摘要：

- 是否需缴费
- 费用状态
- 发票状态

详细金额、发票号、打款日期在财务模块维护。

## Dashboard 打通提醒

Dashboard 后续应能合并显示：

- 新文件已接收但尚未建立递交包。
- 新文件需要培训但尚未建立培训计划。
- 培训计划逾期未完成。
- 递交包超过要求递交期限。
- 已寄中心但未记录 PI 签署。
- 已递交伦理但未记录伦理签收。
- 会审/快审已确认但未创建费用记录。
- 已缴费但未收发票。
- 审查中超过预期但未收到批件/意见函。

## 开发顺序建议

### 1. 项目文件库骨架

先建立项目文件主数据，否则伦理递交包仍然会重复录入文件名称和版本。

### 2. 项目文件触发培训计划

需要培训的项目文件可生成培训计划，并选择授权研究人员。

### 3. 培训记录 Word 生成

培训计划可基于模板生成培训记录/签到表 Word，签署后作为证据收集。

### 4. 递交包引用项目文件

创建递交包时从项目文件库选择文件，并保存快照。

### 5. 递交包生成双递交信

递交信文件清单来自递交包文件快照。

### 6. 财务伦理费台账

会审/快审递交包可一键生成伦理费记录。

### 7. Dashboard 跨模块提醒

把文件、递交、培训、费用四个模块的未闭环事项合并到首页。
