# CRA Portal 产品愿景

> 产品定位：从 CRA Portal 进化为 CRA OS，一个面向临床运营日常工作的 AI Clinical Operations Workspace。

## 使命

帮助 CRA 在一个工作台里完成每天 80% 的高频工作，减少在 Excel、微信、邮件、Word 模板和零散文件之间切换造成的认知负担。

## 核心用户

- 一线 CRA / APM：每天需要跟进项目、中心、待办、Issue、文件、递交、物资和费用。
- 项目协作人员：需要快速了解中心风险、遗留事项、文件状态和待处理动作。

## 产品原则

1. 一个功能如果不能节省 CRA 时间，就暂缓开发。
2. 首页只展示今天需要关注的事情，而不是展示所有数据。
3. 所有数据最终都应该转化为可执行任务、风险提醒或可生成文档。
4. AI 的作用是减少输入、提取信息、生成初稿和发现遗漏，不是增加新的维护负担。
5. 优先支持真实工作流，再补充管理报表。
6. 每个页面都要回答一个问题：CRA 在什么工作场景下会打开它？
7. 系统应该主动提示风险，而不是只被动保存记录。

## 理想的一天

上午打开 Dashboard：

- 看到今天最重要的 3 件事。
- 看到哪个中心风险最高。
- 看到哪些 Issue 快超期。
- 看到哪些文件缺签、缺版本号或影响稽查。
- 看到今天下班前必须完成的动作。

进入中心：

- 看到上次遗留 Issue。
- 看到本次检查 Checklist。
- 看到待签文件、样本待寄、物资待交接、费用待跟进。
- 监查结束后生成 Follow-up Letter 和后续 Task。

## 不做什么

- 不为了功能数量而堆页面。
- 不把 Dashboard 做成纯统计大屏。
- 不让用户重复录入已经存在于报告、表格、模板或邮件中的信息。
- 不在没有清晰验收标准时让 Codex 自由发挥大改。

## 阶段目标

### V1：自己每天可用

- Dashboard 能回答“今天先做什么”。
- Project / Center / Task / Finding 能支撑日常跟进。
- 递交信、物资交接单、费用发票追踪能减少重复 Word/Excel 操作。

### V2：同事也能用

- 数据结构更稳定。
- 模板可配置。
- 操作记录和导出能力完善。
- 关键流程有权限和审计考虑。

### V3：项目组协作

- 项目组视角的风险、中心进度、Issue 关闭率、文件缺口。
- 多角色协作：CRA、APM、PM、CRC 联系人。

### V4：AI CRA Assistant

- 上传 MR / Follow-up Letter / 授权表 / 费用表后自动提取 Issue、Action、Owner、Due Date。
- 自动生成任务。
- 自动发现遗漏、矛盾和高风险中心。

### V5：可商业化产品

- 多租户、权限、审计、模板市场、标准化实施流程。
