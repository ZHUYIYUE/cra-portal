# 中心工作事项：第一版规格

## 目标

把协议、伦理、数据清理、结算等一件完整工作记录为中心级事项，而不是拆成互相脱节的待办。

## 本次范围

- 事项关联项目、中心、项目阶段、类型、优先级和最终节点。
- 每个事项有当前下一步、等待对象、下次催办日期和最后推进日期。
- 支持可编辑步骤与推进记录。
- 当前未完成步骤自动同步为一条带项目和中心信息的待办；完成待办会回写步骤并推进到下一条。
- 从中心详情、独立事项页和 Dashboard 查看同一事项。
- 内置尾款结算、方案偏离递交、SSU 协议、正常值范围、数据清理和预筛选六种步骤模板。

## 不在本次范围

- Excel 批量导入/导出。
- 自动创建多个普通待办或 Findings。
- 财务金额、发票和文件附件上传。
- 自动发送外部消息或邮件。

## 状态与提醒

- `进行中`：CRA 正在推进。
- `等待外部反馈`：必须填写等待对象和建议催办日期。
- `受阻`：需要人工处理阻塞原因。
- `已完成`：不再进入催办清单。

Dashboard 优先显示：催办日期已到期、最终节点临近/逾期，或正在等待外部反馈的事项。

## 验收

1. 可创建一个关联项目和中心的完整事项，并设置下一步与催办日期。
2. 可使用模板生成并调整步骤。
3. 可在中心详情页查看、编辑该中心事项。
4. 到期催办和等待外部反馈事项进入 Dashboard。
5. 每次更新可留下推进记录。
6. 一个事项仅保留一条当前步骤待办；自由创建的待办不受事项更新影响。
