# CRA Portal 部署指南

当前 CRA Portal 是纯静态前端应用，部署在 GitHub Pages：

- 线上地址：https://zhuyiyue.github.io/cra-portal/
- 仓库：https://github.com/ZHUYIYUE/cra-portal
- 本地目录：`D:\办公\项目\cra-portal`

## 前置条件

1. 安装 Git for Windows，或确保 `git` 命令已加入 PATH。
2. 确认 GitHub 凭据可用。推荐使用 GitHub CLI、Credential Manager 或 SSH，不要把 token 写进 remote URL。
3. 确认当前项目目录存在 `.git`。

## 一键部署

运行：

```bat
deploy.bat
```

脚本会：

1. 切换到 `main`
2. 提交并推送源码
3. 切换到 `gh-pages`
4. 从 `main` 同步 `index.html` 和 `static/`
5. 推送 `gh-pages`
6. 切回 `main`

如果没有变更，脚本会跳过空提交。

## 手动部署

```bash
cd "D:\办公\项目\cra-portal"

git checkout main
git add -A
git commit -m "update: 功能描述"
git push origin main

git checkout gh-pages
git checkout main -- index.html static
git add -A
git commit -m "deploy: 功能描述"
git push origin gh-pages

git checkout main
```

## 跨 Mac / Windows 开发

项目已加入 `.gitattributes` 来固定常见文件的换行规则：

- JS/CSS/HTML/SQL/Markdown 使用 LF
- `.bat` / `.cmd` / `.ps1` 使用 CRLF
- Word、Excel、图片、PDF 作为二进制文件处理

如果从 Mac 拉取后再回到 Windows，建议确认：

```bash
git status --short
```

如果出现大量仅换行变化，先不要提交，检查 Git 换行设置和 `.gitattributes` 是否生效。

## GitHub Pages 没更新时

一般等待 1-2 分钟即可。如果页面仍未更新：

1. 打开 GitHub 仓库的 Actions / Pages 页面看构建状态。
2. 确认 `gh-pages` 分支包含最新的 `index.html` 和 `static/`。
3. 浏览器强制刷新：`Ctrl + F5`。
