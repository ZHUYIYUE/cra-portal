# CRA Portal - Supabase 迁移指南

## 概述

本指南将帮助你把 CRA Portal 从 Render PostgreSQL 迁移到 Supabase（永久免费数据库）。

> ⏰ Render SQL 数据库 2026-07-16 到期，请在此前完成迁移。

---

## 步骤 1：创建 Supabase 项目

1. 访问 https://supabase.com 注册/登录
2. 点击 **New Project** 创建项目
3. 填写项目名称（如 `cra-portal`）
4. 设置数据库密码（记下来）
5. 选择离你最近的区域（如 Northeast Asia - Tokyo）
6. 点击 **Create** 等待项目创建完成

## 步骤 2：创建数据库表

1. 在 Supabase Dashboard 左侧菜单点击 **SQL Editor**
2. 点击 **New Query**
3. 打开本项目的 `supabase/schema.sql` 文件，复制全部内容
4. 粘贴到 SQL Editor 中
5. 点击 **Run** 执行

执行成功后会创建以下表：
- `projects` - 项目
- `centers` - 中心
- `tasks` - 待办事项
- `findings` - 监查问题
- `ethics_submissions` - 伦理递交
- `protocol_deviations` - 方案偏离
- `research_staff` - 研究人员
- `startup_tasks` - 启动任务

## 步骤 3：获取 API 密钥

1. 在 Supabase Dashboard 左侧菜单点击 **Project Settings**（齿轮图标）
2. 点击 **API**
3. 记录以下两个值：
   - **Project URL**: `https://xxx.supabase.co`
   - **anon public key**: `eyJxxx...`（很长的字符串）

## 步骤 4：迁移现有数据

在本地终端执行：

```bash
cd Downloads/cra-portal

# 激活虚拟环境
.venv\Scripts\activate

# 设置环境变量（替换为你的实际值）
set SUPABASE_URL=https://xxx.supabase.co
set SUPABASE_KEY=eyJxxx...
set SOURCE_URL=https://cra-portal.onrender.com

# 运行迁移脚本
python migrate_data.py
```

脚本会从线上 API 拉取所有数据并导入 Supabase。
预期输出：
```

✅ 项目: JMKX000197-MPE201 (ab597f8e)
✅ 中心: 01 浙江大学医学院附属第二医院 (1aa5e8b0)
...
✅ 迁移完成！共迁移 32 条记录
```

## 步骤 5：更新 Render 环境变量

1. 打开 Render Dashboard → 你的 CRA Portal 服务
2. 点击 **Environment**
3. 添加/更新以下环境变量：
   - `SUPABASE_URL` = 你的 Supabase Project URL
   - `SUPABASE_KEY` = 你的 Supabase anon key
4. 删除不再需要的环境变量（如旧的数据库连接字符串）
5. 点击 **Save Changes**

## 步骤 6：部署新代码

### 方式 A：Git 推送（推荐）

```bash
cd Downloads/cra-portal
git add -A
git commit -m "迁移到 Supabase 数据库"
git push origin main
```

Render 会自动检测到新提交并重新部署。

### 方式 B：手动部署

在 Render Dashboard 点击 **Manual Deploy** → **Deploy latest commit**

## 常见问题

### Q: Supabase 免费版有什么限制？
- 500MB 数据库存储（CRA Portal 数据量约 15KB，完全够用）
- 50MB 文件存储
- 2GB 月带宽
- 项目 7 天不活动会暂停（可在 Dashboard 手动恢复）

### Q: 忘记数据库密码怎么办？
- Supabase Dashboard → Project Settings → Database → Reset database password

### Q: 迁移后旧数据会丢失吗？
- 不会。迁移脚本只是从线上 API 读取数据写入 Supabase，不影响 Render 上的数据。
- 迁移完成验证无误后，Render SQL 到期也不影响。

### Q: 需要修改前端代码吗？
- 不需要。前端 API 接口完全不变，只是后端从 Render PostgreSQL 换成了 Supabase。
